### 第一步：探测 (Exploration)
1. `await nav(url)` 后**立即** `await get_network_log()`。
2. 若只有框架请求，做一次交互（点筛选/翻页），再查 `get_network_log(only_new=True)`。
3. **API 优先**：发现 API 立即用 `http_json()` 重放，绝不用 DOM 提取。

### 第二步：批量采集
- **Batch 纪律**：所有翻页必须用 `for`/`while` 循环在一步内完成。
- **及时保存**：每页采集完**立即**调用 `await append_items(items)`。
- **系统熔断**：连续 3 次提取数据重复系统将抛异常中断，此时需检查逻辑或等待渲染。

### 第三步：鉴权处理 (API)
- API 返回 401 时 → 检查 network log 中的 `headers` 并完整复刻。
- 或者通过 `await get_cookies()` 获取浏览器凭证后注入 `http_json` 的 `cookies=` 参数。

### 高级搜索与定位技巧
- **论文搜索**：搜学术文档/白皮书加 `filetype:pdf` 或 `site:arxiv.org`。
- **精准匹配**：精确的专有名词使用双引号（如 `"Agent"`）。
- **盲测 API**：不确定 API 的排序/分页参数时，分配一步时间同时测试多个常见 Key (如 `sort`, `pageNo`)。
- **挖掘 JSON**：在写复杂 DOM 提取前，优先检查源码中是否隐藏有 `__NEXT_DATA__` 或 `__NUXT__` 的完整 JSON 脚本。
- **持久化定位**：要在循环中翻页，请先 `await get_selector_from_index(i)` 获取不受刷新影响的 CSS 路径。
