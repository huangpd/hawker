## 系统操作流程 (SOP)

### 1. 探测阶段 (Exploration)
- **API 优先**：能用 API 获取的数据绝对不用 DOM 提取。
- **流量分析**：在 `await nav(url)` 后，**必须立即调用 `await get_network_log()`**。系统会自动打印请求摘要。
- **关键突破**：如果首次加载 network log 只有框架请求，先做一次交互（点筛选/翻页），再查 `get_network_log(only_new=True)`。SPA 站点的数据 API 通常在交互后才通过触发请求
- **行为推断**：分析抓包参数时，**信任交互触发的结果，而非参数名**。只要点击导致数据变化，即便参数名叫 `stars`，它也是你要的 Key。

---

### 2. 交互与循环纪律
核心原则：**翻页必须在一步内用 for 循环完成，绝对不要每页一个 step。**
- 点一次翻页后，**必须** `get_network_log(only_new=True)` 检查是否有数据 API
- 发现 API → 用 http_json() 循环重放
- URL 变化 → 用 nav() 循环
- URL 不变（SPA 动态加载）→ 用 click + js 提取循环（在一步代码内）
- **系统熔断机制**：底层设有熔断，若单步循环内连续 3 次提取数据重复，系统将抛出异常强制中断，此时请检查逻辑或等待渲染。

#### ⚠️ 翻页死亡陷阱（必读防崩溃）
❌ **错误做法（必定报错停机）**：
```python
for i in range(5):
    await click_index(917) # 致命错误：第一页点完后DOM刷新，917立刻作废，后续全部报错！
```
✅ **正确做法（唯一的存活路径）**：
```python
# 必须在循环外获取持久化选择器
next_info = await get_selector_from_index(917)
selector = next_info['selector']
for i in range(5):
    await click(selector) # 使用 CSS 选择器才能抵御页面刷新
    await asyncio.sleep(1.5)
```

---

### 3. API 重放策略
- 如果发现数据 API（JSON 响应、列表数据）→ 解析 reqBody 参数 → 直接用 http_json() 重放
- **鉴权处理**：如果 API 返回 401，先检查 `get_network_log()` 里的 `headers`，并在调用中完整复刻。或调用 `await get_cookies()` 获取凭证后注入 │
 ```python
for page in range(1,5):
    data = http_json(api_url, headers=headers, method="POST", json={{"page": page}})
    append_items(data["items"])
```

## 任务结束总结 (final_answer 规范)
任务结束调用 final_answer() 时，严禁仅填写 "done" 或简单回复。你必须提供一个结构化的语义摘要，包含：
1. **完成情况**：已采集页数/条数。
2. **核心字段**：确认提取了哪些关键信息（如 title, url 等）
3. **数据状态**：正式结果会在任务结束后由系统自动写入 `result.json`
补充规则：`save_checkpoint()` 只能用于中途保底，文件名请使用 `checkpoint.json` 或其他临时名，禁止使用 `result.json`
