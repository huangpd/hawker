## 系统操作流程 (SOP)

### 1. 探测阶段 (Exploration)
- **API 优先**：能用 API 获取的数据绝对不用 DOM 提取。
- **流量分析**：在 `await nav(url)` 后，**必须立即调用 `await get_network_log()`**。系统会自动打印请求摘要。
- **关键突破**：如果首次加载 network log 只有框架请求，先做一次交互（点筛选/翻页），再查 `get_network_log(only_new=True)`。SPA 站点的数据 API 通常在交互后才通过 POST 触发。
- **行为推断**：分析抓包参数时，**信任交互触发的结果，而非参数名**。只要点击导致数据变化，即便参数名叫 `stars`，它也是你要的 Key。

### 2. 交互与循环纪律
- **二转策略 (The Two-Strike Rule)**：**禁止连续 2 次以上执行完全相同的操作。** 若点击某索引 Observation 无变化，必须切换方法：换更深层元素点击、使用 `js()` 脚本强制点击、或直接盲测 API。
- **Batch 纪律**：所有翻页必须在一步代码内用 `for`/`while` 循环完成。每采集完一页必须立即调用 `await append_items(items)`。
- **系统熔断机制**：底层设有熔断，若单步循环内连续 3 次提取数据重复，系统将抛出异常强制中断，此时请检查逻辑或等待渲染。

#### ⚠️ 翻页死亡陷阱（必读防崩溃）
❌ **错误做法（必定报错停机）**：
```python
for i in range(5):
    await click_index(917) # 致命错误：第一页点完后DOM刷新，917立刻作废，后续全部报错！
```
✅ **正确做法（唯一的存活路径）**：
```python
# 必须在循环外获取持久化选择器
next_info = await get_selector_from_index(917)
selector = next_info['selector']
for i in range(5):
    await click(selector) # 使用 CSS 选择器才能抵御页面刷新
    await asyncio.sleep(1.5)
```

### 3. API 重放策略
- 发现 API 后，直接用 `http_json()` 构造请求。
- **鉴权处理**：如果 API 返回 401，先检查 `get_network_log()` 里的 `headers`，并在调用中完整复刻。或调用 `await get_cookies()` 获取凭证后注入。

### 4. 高级搜索与定位技巧
- **论文搜索**：搜学术文档/白皮书加 `filetype:pdf` 或 `site:arxiv.org`。
- **精准匹配**：精确的专有名词使用双引号（如 `"Agent"`）。
- **盲测 API**：不确定 API 的排序/分页参数时，分配一步时间同时测试多个常见 Key (如 `sort`, `pageNo`)。
- **挖掘 JSON**：在写复杂 DOM 提取前，优先检查源码中是否隐藏有 `__NEXT_DATA__` 或 `__NUXT__` 的完整 JSON 脚本。
- **持久化定位**：要在循环中翻页，请先 `await get_selector_from_index(i)` 获取不受刷新影响的 CSS 路径。
